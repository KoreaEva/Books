## 01.lotto_gernerator.py

### 주석문
```# 요구사항 정리
# 1) 1부터 45까지 숫자 중에서 중복 없이 6개를 뽑는다.
# 2) 결과는 오름차순으로 정렬한다.
# 3) 사용자에게 "몇 게임을 만들지"와 "시드값"을 입력받는다.
#    - 시드를 비우면 무작위로 생성한다.
#    - 같은 시드를 주면 같은 결과를 재현할 수 있다.
# 4) 여러 게임을 만들 때는 각 게임마다 조금씩 다른 시드를 써서 결과를 다양화한다.
# 5) 잘못된 입력(문자 입력 등)은 기본값으로 처리하고 안내 메시지를 출력한다.
# 6) 한 파일로 구성하며, 함수 generate_ticket()과 main()만으로 단순하게 유지한다.
```

### prompt
> 파이썬으로 로또 번호 생성기를 구현하되, generate_ticket(seed=None) 함수는 1부터 45 사이 수에서 중복 없이 6개를 뽑아 오름차순 리스트로 반환하는 기능으로 만든다. main()은 사용자에게 게임 수와 시드를 입력받아 결과를 출력하고 시드가 없으면 무작위, 있으면 재현 가능하도록 한다. 여러 게임일 경우 base_seed+(i-1)로 시드를 다르게 하며, 잘못된 입력은 기본값(게임 수=1, 시드=None)으로 처리해 안내 메시지를 출력한다.

## 06.todo-manager.py

### 주석문
```
# 요구사항 정리
# 1) 작업 추가(add): 작업 내용과 함께 선택적으로 마감일(due), 우선순위(priority)를 입력받는다.
# 2) 작업 조회(list): 전체/미완료/완료만 필터링하여 번호와 함께 출력한다.
# 3) 작업 완료(complete): 번호로 선택해 완료 처리한다(완료 시각 기록).
# 4) 작업 삭제(delete): 번호로 선택해 삭제한다.
# 5) 보조 기능: 완료 항목 일괄 삭제(clear done), 파일 저장/불러오기 자동화
# 6) 하나의 파일로 구성: load/save, add/list/complete/delete/clear_done, main()으로 단순하게 유지한다.
# 7) 데이터는 todos.json(JSON)으로 저장한다(표준 라이브러리만 사용).
```

### prompt
> 파이썬으로 파일 기반 To-Do CLI 앱을 구현한다. todos.json에 작업을 저장하고, add/list/complete/delete/clear-done 기능을 제공한다. 작업은 id, text, created_at, due(optional), priority(1~5, default 3), done(bool), done_at(optional) 속성을 갖는다. main()은 텍스트 메뉴로 사용자 입력을 받아 각 기능을 호출한다. 입력 검증과 예외 처리, 깔끔한 출력 서식을 포함하라. 표준 라이브러리만 사용할 것.

## 07.momorize_word.py

### 주석문
```
# 요구사항 정리
# 1) 단어 추가(add): word(영어), meaning(국문), example(선택)을 입력받아 저장한다.
# 2) 복습 주기(Leitner): box 1~5에 따라 다음 복습일을 계산한다.
#    예) box1=1일, box2=2일, box3=4일, box4=7일, box5=14일 후
# 3) 퀴즈(review/quiz): 오늘 복습 대상 우선 출제, 정답이면 box+1(최대5), 오답이면 box=1
# 4) 목록(list): 전체/오늘 복습 대상을 표로 출력한다.
# 5) 삭제(delete): id로 삭제한다. 저장은 vocab.json을 사용한다.
# 6) 하나의 파일로 구성: 데이터 load/save와 add/list/review/delete, main()으로 단순하게 유지한다(표준 라이브러리만 사용).
# 7) 단어는 입력하자 마자 퀴즈에 사용할 수 있다. 
```

### prompt
> 파이썬으로 CLI 영어 단어 암기 앱을 구현하라. vocab.json 파일에 단어를 저장하고 라이트너(Leitner) 상자(1~5)에 따른 복습 주기로 '오늘 복습할 항목'을 우선 퀴즈로 출제한다.
데이터 모델은 id, word, meaning, example(optional), box(1~5), created_at, last_review(optional),
next_review(optional), correct_streak, wrong_streak을 포함한다.
정답 시 box+1(최대 5), 오답 시 box=1로 리셋하고 next_review를 재계산한다. add/list(review_due_only)/quiz/delete 기능과 입력 검증, 보기 좋은 출력 서식을 포함한다. 표준 라이브러리만 사용할 것.

## 08.meeting_summary.py

### 주석문
```
# Whisper(STT) 기능 구현 요구사항
# 1) 사용자로부터 오디오 파일 경로를 입력받는다.
# 2) 위스퍼(Whisper) 모델을 사용해 음성을 받아쓴다. 언어는 자동으로 감지하며 모델 크기는 기본으로 small을 사용한다.
# 3) 파일이 없거나 Whisper/FFmpeg 미설치 시 예외를 처리하고 안내 메시지를 출력한다.
# 4) transcribe() 함수 단일 구조로 작성하며 인식된 결과를 텍스트 파일로 저장한다.
# 5) 성공하면 받아쓴 텍스트를 반환하고, 실패하면 사용자 친화적인 오류 메시지를 제공한다.
# 6) 저장되는 파일명은 '날짜 + 일련번호' 형식으로 자동 생성한다.
```

### prompt

> 사용자가 음성 파일의 경로를 입력하면 위스퍼(Whisper) 모델을 이용해 음성 파일에서 텍스트를 추출하고, 결과를 텍스트 파일로 저장하는 코드를 작성한다. 파일명은 '날짜 + 일련번호' 형식으로 자동 생성한다.
